## 객관식 (3문제)

### Q1. ECS 클러스터 구성에 대한 설명 중 **오류**가 있는 것은?  
- A) 클러스터는 ECS 서비스 실행을 위한 리소스 집합으로, 여러 태스크를 관리함  
- B) 클러스터 생성 시 `FARGATE` 또는 `EC2` 유형을 선택해야 함  
- C) 태스크는 컨테이너를 포함하는 단위로, 서비스는 태스크를 실행하는 로직을 제공함  
- D) 클러스터는 EC2 인스턴스와 Fargate를 병행하여 구성할 수 없음  

### Q2. EKS의 컨트롤 플레인에 대한 설명으로 **맞는 것은**?  
- A) 노드 그룹을 직접 관리하는 기능을 제공함  
- B) AWS Managed Node Group을 사용해 자동화된 노드 관리를 제공함  
- C) 컨트롤 플레인은 Kubernetes API 서버, etcd, kubelet으로 구성됨  
- D) EKS의 컨트롤 플레인은 사용자가 직접 관리해야 함  

### Q3. Fargate 자동 확장에 대한 설명 중 **틀린 것은**?  
- A) 요청량에 따라 자동으로 CPU/메모리 리소스를 조정함  
- B) 최소/최대 인스턴스 수를 설정할 수 있음  
- C) Fargate는 수동으로 스케일링을 지원하지 않음  
- D) Auto Scaling 그룹과 연동해 탄력적인 리소스 관리를 제공함
