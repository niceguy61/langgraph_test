# Week 1 Day 4 복습 퀴즈

## 객관식 (3문제)

### Q1. 다음 중 EBS 볼륨 유형 중 **고성능 애플리케이션**에 최적화된 유형은 무엇인가요?  
- A) gp3  
- B) io2  
- C) st1  
- D) sc1  

### Q2. EBS 스냅샷의 주요 목적은 무엇인가요?  
- A) 인스턴스의 실시간 상태를 저장  
- B) 데이터 복구 및 백업  
- C) 네트워크 트래픽 제어  
- D) 보안 그룹 규칙 생성  

### Q3. 보안 그룹에서 다음 중 **기본적으로 허용되지 않는** 트래픽은 무엇인가요?  
- A) SSH 접근 (22번 포트)  
- B) HTTP 접근 (80번 포트)  
- C) HTTPS 접근 (443번 포트)  
- D) 0.0.0.0/0에서의 모든 트래픽

## OX 문제 (2문제)

### Q4. EFS는 여러 인스턴스가 동시에 파일 시스템에 접근할 수 있도록 설계된 분산 파일 시스템입니다.  
(O / X)  

### Q5. Instance Store는 인스턴스가 종료되면 데이터가 사라지는 **일시적 저장소**입니다.  
(O / X)  

---

## 정답

| 문제 | 정답 | 해설 |
|-----|------|---|
| Q1 | **A) gp3** | gp3는 고성능 I/O를 제공하는 EBS 볼륨 유형으로, 랜덤 읽기/쓰기 작업에 최적화되어 있습니다. |
| Q2 | **B) 데이터 복구 및 백업** | EBS 스냅샷은 백업 및 재해 복구를 위한 주요 도구로, 백그라운드에서 볼륨 데이터를 백업합니다. |
| Q3 | **D) 0.0.0.0/0에서의 모든 트래픽** | 보안 그룹은 기본적으로 모든 트래픽을 거부하며, 특정 IP 또는 포트만 명시적으로 허용해야 합니다. |
| Q4 | **O** | EFS는 클라우드 기반의 분산 파일 시스템으로, 여러 EC2 인스턴스가 동시에 파일에 접근할 수 있도록 설계되었습니다. |
| Q5 | **O** | Instance Store는 인스턴스 종료 시 데이터가 소멸하는 일시적 저장소로, EBS와 달리 백업이 필요합니다. |
