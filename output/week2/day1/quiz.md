# Week 2 Day 1 복습 퀴즈

## 객관식 (3문제)

### Q1. VPC CIDR 주소 공간의 주요 역할은 무엇인가요?  
- A) 보안 그룹의 규칙 정의  
- B) 가상 네트워크의 IP 범위 정의  
- C) 로그 수집을 위한 식별자  
- D) 인스턴스의 CPU 사용량 모니터링  

### Q2. 다음 중 퍼블릭 서브넷과 프라이빗 서브넷의 주요 차이점은 무엇인가요?  
- A) 퍼블릭 서브넷은 SSH 접근이 불가능함  
- B) 퍼블릭 서브넷은 인터넷 게이트웨이로 연결되어야 함  
- C) 프라이빗 서브넷은 VPC 내부에만 존재함  
- D) 프라이빗 서브넷은 라우팅 테이블에 반드시 포함되어야 함  

### Q3. NAT 게이트웨이와 NAT 인스턴스의 주요 차이점은 무엇인가요?  
- A) NAT 게이트웨이는 EC2 인스턴스를 필요로 하지 않음  
- B) NAT 인스턴스는 고가용성을 자동 제공함  
- C) NAT 게이트웨이는 네트워크 트래픽을 감시할 수 있음  
- D) NAT 인스턴스는 VPC 내부에만 배포 가능함

### Q4. 퍼블릭 서브넷은 항상 인터넷 게이트웨이로 연결되어 있어야 합니다. (O/X)  
### Q5. NAT 게이트웨이는 인바운드 트래픽을 처리할 수 있습니다. (O/X)

## 정답

| 문제 | 정답 | 해설 |
|-----|------|---|
| Q1 | **B** | VPC CIDR은 가상 네트워크의 IP 범위를 정의하는 데 사용됩니다. 보안 그룹은 규칙, 로그 수집은 CloudWatch, 인스턴스 모니터링은 CloudWatch Metrics에 해당합니다. |
| Q2 | **B** | 퍼블릭 서브넷은 인터넷 게이트웨이로 연결되어 외부 인터넷에 접근할 수 있습니다. 프라이빗 서브넷은 내부 네트워크만 접근 가능하며, 라우팅 테이블 구성에 따라 외부 접근이 제어됩니다. |
| Q3 | **A** | NAT 게이트웨이는 AWS가 관리하는 서비스로 EC2 인스턴스가 필요하지 않으며, NAT 인스턴스는 EC2 인스턴스를 사용해야 합니다. NAT 인스턴스는 고가용성과 확장성이 낮습니다. |
| Q4 | **O** | 퍼블릭 서브넷은 인터넷 게이트웨이로 연결되어 외부 인터넷에 접근할 수 있습니다. 프라이빗 서브넷은 인터넷 게이트웨이와 연결되지 않아 외부 접근이 제한됩니다. |
| Q5 | **X** | NAT 게이트웨이는 외부에서 내부로의 트래픽(인바운드)을 처리할 수 없습니다. NAT 인스턴스는 인바운드 트래픽을 처리할 수 있지만, NAT 게이트웨이는 단방향(아웃바운드)만 지원합니다. |
