# Week 2 Day 2 복습 퀴즈

## 객관식 (3문제)

### Q1. VPC 피어링의 주요 목적은 무엇인가요?  
- A) VPC 간의 공개 인터넷 연결 제공  
- B) VPC 간의 프라이빗 네트워크 연결 구축  
- C) 서버 간의 라우팅 테이블 자동 설정  
- D) 서버에 대한 보안 그룹 정책 적용  

### Q2. 트랜지트 게이트웨이의 주요 기능은 무엇인가요?  
- A) VPC 간의 직접 연결을 위한 라우팅 테이블 제공  
- B) 여러 VPC와 하이브리드 네트워크를 중앙 집중식으로 연결  
- C) VPC 내부 서버에 대한 방화벽 설정  
- D) VPC의 IP 주소 범위를 동적으로 확장  

### Q3. VPC 게이트웨이 엔드포인트와 인터페이스 엔드포인트의 주요 차이점은 무엇인가요?  
- A) 게이트웨이 엔드포인트는 S3 서비스만 지원하고, 인터페이스 엔드포인트는 모든 서비스 지원  
- B) 게이트웨이 엔드포인트는 VPC 내부 네트워크를 통해 서비스 접근, 인터페이스 엔드포인트는 인터넷을 통해 접근  
- C) 게이트웨이 엔드포인트는 전용 IP 주소를 사용하고, 인터페이스 엔드포인트는 공용 IP 주소 사용  
- D) 게이트웨이 엔드포인트는 네트워크 트래픽을 라우팅하고, 인터페이스 엔드포인트는 보안 정책 적용  

## OX 문제 (2문제)

### Q4. VPC 피어링 연결을 설정할 때 트랜지트 게이트웨이가 필수적으로 필요합니다.  
(O / X)

### Q5. Direct Connect는 AWS와의 전용 네트워크 인터페이스를 통해 연결을 구성합니다.  
(O / X)

## 정답

| 문제 | 정답 | 해설 |
|-----|------|---|
| Q1 | B | VPC 피어링은 두 VPC 간의 프라이빗 네트워크 연결을 제공합니다. 공개 인터넷 연결은 NAT 게이트웨이 또는 라우팅 테이블 설정에 해당합니다. |
| Q2 | B | 트랜지트 게이트웨이는 여러 VPC와 하이브리드 네트워크를 중앙 집중식으로 연결하는 역할을 합니다. |
| Q3 | B | 게이트웨이 엔드포인트는 VPC 내부 네트워크를 통해 AWS 서비스에 접근하고, 인터페이스 엔드포인트는 인터넷을 통해 접근합니다. |
| Q4 | X | VPC 피어링은 트랜지트 게이트웨이 없이도 두 VPC 간의 직접 연결이 가능합니다. 트랜지트 게이트웨이는 다른 네트워크 구성에 사용됩니다. |
| Q5 | O | Direct Connect는 AWS와의 전용 네트워크 인터페이스를 통해 고대역폭의 연결을 제공합니다. |
