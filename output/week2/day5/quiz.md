# Week 2 Day 5 복습 퀴즈

## 객관식 (3문제)

### Q1. S3 스토리지 클래스 중, **자주 액세스되는 데이터**에 적합한 스토리지 클래스는 무엇인가요?  
- A) Glacier  
- B) Glacier Deep Archive  
- C) Standard  
- D) One Zone-Infrequent Access  

### Q2. S3 수명주기 정책에서 **객체 삭제**를 설정할 수 있는가요?  
- A) 예, 수명주기 정책으로 객체를 삭제할 수 있습니다.  
- B) 예, 수명주기 정책으로 객체를 삭제할 수 있지만, 객체가 90일 이상 존재해야 합니다.  
- C) 아니요, 수명주기 정책으로 객체를 삭제할 수 없습니다.  
- D) 아니요, 수명주기 정책은 객체를 삭제할 수 없고, 오직 데이터를 아카이브할 수 있습니다.  

### Q3. CloudFront 배포 시, 다음 중 **원천(Origin)**으로 사용할 수 없는 것은 무엇인가요?  
- A) S3 버킷  
- B) EC2 인스턴스  
- C) 라우팅 정책이 설정된 Route 53 호스팅 존  
- D) Amazon DynamoDB 테이블

### Q4. Route 53 라우팅 정책 중 **Weighted Routing**은 트래픽을 가중치에 따라 분배할 수 있는 기능입니다.  
(O/X)  

### Q5. Global Accelerator는 AWS의 글로벌 네트워크를 기반으로 트래픽을 최적화하여 지연을 줄이는 서비스입니다.  
(O/X)

## 정답

| 문제 | 정답 | 해설 |
|-----|------|---|
| Q1 | **C) Standard** | Standard 스토리지 클래스는 빈도가 높고 액세스가 빈번한 데이터에 적합합니다. Glacier는 저빈도 데이터에, One Zone-Infrequent Access는 단일 존에 저장되는 저빈도 데이터에 사용됩니다. |
| Q2 | **A) 예, 수명주기 정책으로 객체를 삭제할 수 있습니다.** | S3 수명주기 정책은 객체를 삭제하거나, 데이터를 아카이브( Glacier로 이동)할 수 있습니다. 삭제 시 객체는 즉시 삭제되며, 아카이브 시는 90일 이상 경과해야 합니다. |
| Q3 | **D) Amazon DynamoDB 테이블** | CloudFront의 원천으로는 S3 버킷, EC2 인스턴스, Elastic Load Balancing, Route 53 호스팅 존 등이 가능하지만, DynamoDB 테이블은 원천으로 설정할 수 없습니다. |
| Q4 | **O** | Weighted Routing은 트래픽을 가중치에 따라 분배하여 실험 또는 A/B 테스트에 유용합니다. 예를 들어, 70%의 트래픽을 서버 A로, 30%를 서버 B로 분배할 수 있습니다. |
| Q5 | **O** | Global Accelerator는 AWS의 글로벌 네트워크를 기반으로 트래픽을 최적화하여 지연을 줄이고, 사용자에게 가까운 지역의 엔드포인트로 전달합니다. |
