# Week 2 Day 4 복습 퀴즈

## 객관식 (3문제)

### Q1. 다음 중 S3 버킷 정책과 ACL의 차이점에 대한 설명으로 옳은 것은?  
- A) ACL은 버킷 수준에서만 적용되고, 버킷 정책은 객체 수준에서 적용됨  
- B) 버킷 정책은 AWS 서비스 간 권한을 제어하고, ACL은 사용자별 권한을 제어함  
- C) ACL은 모든 사용자에게 동일한 권한을 부여하고, 버킷 정책은 특정 사용자에게만 권한을 부여함  
- D) 버킷 정,책은 S3 Glacier 전용이며, ACL은 S3 Standard에만 적용됨  

### Q2. SSE-KMS 암호화의 주요 특징으로 옳은 것은?  
- A) AWS Key Management Service를 사용하여 암호화 키를 관리함  
- B) 암호화 키는 클라이언트가 직접 생성하고 관리함  
- C) 암호화는 객체 생성 시 자동으로 수행되지 않음  
- D) 암호화 키는 S3 버킷 내에서 저장됨  

### Q3. S3 버전 관리 기능에 대한 설명 중 잘못된 것은?  
- A) 버전 관리가 활성화되면 객체 삭제 시 이전 버전이 자동으로 복구됨  
- B) 버전 관리가 활성화된 버킷에서 MFA Delete를 사용할 수 있음  
- C) 버전 관리가 활성화된 버킷에서 객체를 삭제할 때는 항상 MFA를 요구함  
- D) 버전 관리가 비활성화된 버킷에서 객체를 삭제하면 영구 삭제됨  

## OX 문제 (2문제)

### Q4. S3 Object Lock이 활성화되면 객체는 영원히 삭제 또는 수정할 수 없음. (O/X)  
### Q5. S3 버킷 정책은 ACL보다 더 세밀한 권한 제어가 가능함. (O/X)  

---

## 정답

| 문제 | 정답 | 해설 |
|-----|------|---|
| Q1 | B | 버킷 정책은 AWS 서비스 간 권한을 제어하고, ACL은 사용자별 권한을 제어함. 버킷 정책은 JSON 형식으로 정의되며, ACL은 텍스트 기반의 권한 설정을 제공함. |
| Q2 | A | SSE-KMS는 AWS KMS를 사용하여 암호화 키를 관리함. 키는 AWS 관리되는 키(AWS Managed Key) 또는 사용자 생성 키(Customer Managed Key)로 구성될 수 있음. |
| Q3 | C | 버전 관리가 활성화된 버킷에서 객체 삭제 시 MFA Delete를 사용할 수 있지만, 모든 삭제 요청에 MFA가 필요하지 않음. 단일 객체 삭제는 MFA 없이 가능함. |
| Q4 | X | S3 Object Lock은 특정 기간 동안 객체를 보호할 수 있지만, 해지 기간이 지나면 삭제 또는 수정이 가능함. 영원히 삭제할 수 없는 기능은 아님. |
| Q5 | O | 버킷 정책은 JSON 형식으로 정의되어 특정 사용자, 서비스, 지역 등 세밀한 권한을 설정할 수 있음. ACL은 텍스트 기반으로 제한적인 권한 설정만 가능함. |
